# Telegram MVP-бот для поиска тиммейта

Теперь бот поддерживает **группы пользователей**: заявки видны только участникам одной группы.

## Что умеет бот

1. Пользователь создаёт/вступает в группу:
   - `/create_group <название>`
   - `/join_group <код>`
2. Создаёт заявку (`/new_request` или кнопка):
   - выбирает игру (кнопками или текстом);
   - указывает время.
3. Бот рассылает заявку только участникам той же группы.
4. Получатели отвечают кнопками:
   - `Могу играть`
   - `Не могу`
5. Создатель заявки видит статусы ответов (`/my_requests`).

## Кнопки

- Постоянное меню:
  - `➕ Новая заявка`
  - `📋 Мои заявки`
  - `👥 Моя группа`
- Inline-кнопки для быстрого выбора популярных игр.
- Inline-кнопки ответа на заявку.

## Структура

- `bot.py` — обработчики Telegram-команд, FSM, рассылка, кнопки, групповой доступ.
- `storage.py` — SQLite-хранилище пользователей, групп, заявок и ответов.

## Что реализовано

- Пошаговый сценарий (FSM) создания заявки.
- Хранение пользователей, заявок и ответов в SQLite.
- Inline-кнопки для быстрых ответов.
- Отправка обновлений создателю заявки при каждом новом ответе.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# впишите BOT_TOKEN
python bot.py
```

## Команды

- `/start` — регистрация и помощь.
- `/create_group <название>` — создать группу и получить код приглашения.
- `/join_group <код>` — вступить в группу.
- `/my_group` — посмотреть текущую группу и код.
## Команды бота

- `/start` — регистрация и справка.
- `/new_request` — создать новую заявку.
- `/my_requests` — список последних заявок и просмотр ответов.

## Ограничения MVP

- Polling вместо webhook.
- Время игры — свободный текст.
- Участник должен написать боту `/start`, чтобы получать личные сообщения.
- Используется polling (не webhook).
- Время игры вводится свободным текстом.
- Пользователь получает рассылки только после того, как хотя бы раз нажал `/start`.
